## ADDED Requirements

### Requirement: 도서 등록
시스템은 인증된 사용자가 새 도서를 등록할 수 있어야 한다. 필수 필드: 제목, 저자. 선택 필드: ISBN, 출판사, 출판일, 설명, 표지 이미지 URL, 카테고리.

#### Scenario: 필수 필드만으로 도서 등록
- **WHEN** 사용자가 제목과 저자를 입력하고 등록 버튼을 클릭
- **THEN** 도서가 생성되고 도서 상세 페이지로 이동

#### Scenario: 필수 필드 누락 시 등록 실패
- **WHEN** 사용자가 제목 없이 등록 시도
- **THEN** 유효성 검증 오류 메시지가 표시되고 도서는 생성되지 않음

### Requirement: 도서 목록 조회
시스템은 등록된 도서를 페이지네이션된 목록으로 표시해야 한다. 기본 정렬은 최신 등록순이며, 한 페이지당 20권을 표시한다.

#### Scenario: 도서 목록 페이지 접근
- **WHEN** 사용자가 /books 페이지에 접근
- **THEN** 도서 목록이 테이블 형태로 표시되고 제목, 저자, 카테고리, 대출 상태가 표시됨

#### Scenario: 페이지네이션
- **WHEN** 도서가 20권 초과 등록되어 있을 때
- **THEN** 페이지 하단에 페이지네이션 컨트롤이 표시됨

### Requirement: 도서 상세 조회
시스템은 개별 도서의 상세 정보를 표시해야 한다.

#### Scenario: 도서 상세 페이지 접근
- **WHEN** 사용자가 /books/:id 페이지에 접근
- **THEN** 도서의 모든 정보(제목, 저자, ISBN, 출판사, 출판일, 설명, 표지, 카테고리, 태그, 대출 상태)가 표시됨

### Requirement: 도서 수정
시스템은 인증된 사용자가 도서 정보를 수정할 수 있어야 한다.

#### Scenario: 도서 정보 수정
- **WHEN** 사용자가 도서 수정 페이지에서 정보를 변경하고 저장
- **THEN** 변경사항이 반영되고 도서 상세 페이지로 이동

### Requirement: 도서 삭제
시스템은 인증된 사용자가 도서를 삭제할 수 있어야 한다. 대출 중인 도서는 삭제할 수 없다.

#### Scenario: 도서 삭제
- **WHEN** 사용자가 대출 중이 아닌 도서의 삭제 버튼을 클릭하고 확인
- **THEN** 도서가 삭제되고 도서 목록 페이지로 이동

#### Scenario: 대출 중인 도서 삭제 시도
- **WHEN** 사용자가 대출 중인 도서의 삭제를 시도
- **THEN** "대출 중인 도서는 삭제할 수 없습니다" 오류 메시지 표시
