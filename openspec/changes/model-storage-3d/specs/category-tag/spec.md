## MODIFIED Requirements

### Requirement: 카테고리 CRUD
시스템은 3D 모델 분류를 위한 카테고리를 관리할 수 있어야 한다. 카테고리는 단일 계층 구조(부모-자식)를 지원한다. 모델이 연결된 카테고리는 삭제할 수 없다.

#### Scenario: 카테고리 생성
- **WHEN** 사용자가 카테고리 이름 입력 후 생성
- **THEN** 카테고리가 생성되고 목록에 표시됨

#### Scenario: 모델이 있는 카테고리 삭제 시도
- **WHEN** 모델이 연결된 카테고리 삭제 시도
- **THEN** "해당 카테고리에 모델이 존재합니다" 오류 표시

### Requirement: 태그 관리
시스템은 모델에 자유 태그를 부여할 수 있어야 한다. 태그는 modelTags 테이블에 직접 text로 저장된다.

#### Scenario: 모델에 태그 추가
- **WHEN** 사용자가 모델 업로드/수정 시 태그 입력
- **THEN** 태그가 modelTags 테이블에 저장됨

#### Scenario: 태그 기반 필터링
- **WHEN** 사용자가 목록에서 태그 선택
- **THEN** 해당 태그가 있는 모델만 필터링됨
