## ADDED Requirements

### Requirement: 모델 업로드
시스템은 인증된 사용자가 3D 모델 파일(STL, OBJ, GLTF, GLB)을 업로드할 수 있어야 한다. 필수: 파일, 이름, 카테고리. 선택: 설명, 태그.

#### Scenario: 모델 파일 업로드 성공
- **WHEN** 사용자가 STL 파일을 선택하고 이름/카테고리를 입력한 후 업로드
- **THEN** 파일이 저장되고 모델 상세 페이지로 이동됨

#### Scenario: 지원되지 않는 파일 형식
- **WHEN** 사용자가 .png 파일을 업로드 시도
- **THEN** "지원되지 않는 파일 형식입니다" 오류 표시

#### Scenario: 파일 크기 초과
- **WHEN** 사용자가 50MB 초과 파일 업로드 시도
- **THEN** "파일 크기가 50MB를 초과합니다" 오류 표시

### Requirement: 모델 목록 조회
시스템은 업로드된 모델을 페이지네이션된 목록/그리드로 표시해야 한다.

#### Scenario: 모델 목록 페이지
- **WHEN** 사용자가 /models 페이지 접근
- **THEN** 모델 카드 그리드가 표시되고 이름, 파일타입, 카테고리, 태그가 표시됨

### Requirement: 모델 상세 조회
시스템은 모델의 전체 정보와 3D 미리보기를 표시해야 한다.

#### Scenario: 모델 상세 페이지
- **WHEN** 사용자가 /models/:id 접근
- **THEN** 모델 메타데이터(이름, 설명, 파일타입, 파일크기, 카테고리, 태그)와 3D 미리보기가 표시됨

### Requirement: 모델 수정
시스템은 모델 메타데이터(이름, 설명, 카테고리, 태그)를 수정할 수 있어야 한다. 파일 자체는 수정 불가.

#### Scenario: 모델 메타데이터 수정
- **WHEN** 사용자가 모델 수정 페이지에서 이름을 변경하고 저장
- **THEN** 변경사항이 반영되고 모델 상세 페이지로 이동

### Requirement: 모델 삭제
시스템은 모델과 해당 파일을 삭제할 수 있어야 한다.

#### Scenario: 모델 삭제
- **WHEN** 사용자가 삭제 버튼을 클릭하고 확인
- **THEN** DB 레코드와 파일 시스템의 파일이 모두 삭제됨

### Requirement: 모델 파일 다운로드
시스템은 모델 파일을 다운로드할 수 있어야 한다.

#### Scenario: 파일 다운로드
- **WHEN** 사용자가 다운로드 버튼 클릭
- **THEN** 적절한 Content-Type과 Content-Disposition 헤더로 파일이 다운로드됨
